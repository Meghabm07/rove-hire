# ROVE Hire

ROVE Hire is a Next.js internal recruitment tool for managing candidates from resume intake through offer generation and final hiring decisions. It includes HR auth, job openings, candidate intake, public magic-link applications, interviews, offer/NDA PDF generation, and hired/rejected transitions.

## Tech Choices

The frontend uses Next.js App Router with server actions for form flows. The backend lives in `server/` as an Express API connected to Postgres via `pg`; Postgres is a good fit here because candidate, job, and timeline data are relational and need durable querying as the product grows.

Docker Compose runs three services: `web`, `server`, and `postgres`. Resumes and generated PDFs are stored by the backend under `server/uploads` locally and in a Docker volume in Compose; at scale this should move to object storage such as S3, Cloudflare R2, or Supabase Storage.

For a free deployment path, use Render or Railway for the Dockerized API, Vercel or Render for the frontend, and a free Neon or Supabase Postgres database. Set `DATABASE_URL` on the backend, `INTERNAL_API_URL` on the frontend for server-to-server calls, and `NEXT_PUBLIC_SERVER_URL` so browser download links point at the public API host.

## Run Locally

```bash
npm install
npm run dev
```

In another terminal, run the backend locally:

```bash
cd server
npm install
npm run dev
```

Or run the full stack with Docker:

```bash
docker compose up --build
```

The frontend expects the backend at `INTERNAL_API_URL`, defaulting to `http://localhost:4000/api`.

## PDF Generation

Offer letters and NDAs are generated by the backend as lightweight PDF files from structured templates. For a larger production system I would switch this to a richer template renderer with versioned templates, preview, and audit trails.
